ZOWI BASE v2

The default firmware for Zowi. Developed by BQ (Anita de Prado, Jose Alberca, Javier Isabel, Juan Gonzalez, Irene Sanz, December 2015). Released under GPL.

Overview

ZOWI_BASE_v2 is the main firmware that ships with every Zowi robot. It implements a 5-mode state machine accessible via the back buttons, plus a full serial command protocol for remote control via the ZowiApp (Bluetooth).

Boot Sequence

1. Factory Name Check

if (EEPROM.read(5) == '$') {
    EEPROM.put(5, '#');
    EEPROM.put(6, '\0');
    zowi.putMouth(culito);
    while(true) delay(1000);
}

If the EEPROM name is $ (factory marker), Zowi writes # as the first name and enters an infinite loop showing a culito mouth. This is the factory state — the robot ships like this and only exits this loop when the user sets a name via the app.

2. Unbaptized Greeting

if (EEPROM.read(5) == '#') {
    zowi.jump(1, 700);
    zowi.shakeLeg(1, T, 1);
    zowi.putMouth(smallSurprise);
    zowi.swing(2, 800, 20);
    zowi.home();
}

If the name is still # (never set by the user), Zowi performs an extended greeting animation. Once the user sets a name via the app, this greeting is skipped on subsequent boots.

3. Boot Animations

After the name check, Zowi plays a "littleUuh" mouth animation, a smile with S_happy sound, and if unbaptized: jump, shake leg, and swing.

4. Idle Loop

After setup, Zowi enters Mode 0 (Idle). Every 80 seconds of inactivity, it falls asleep with a dream animation and snoring sounds.

Mode Logic

Mode 0 — Idle

Zowi stands still with a happy mouth. After 80 seconds of inactivity, it falls asleep: leans forward, plays a 4-frame dream mouth animation with soft tones, then shows lineMouth and plays S_cuddly. Any button press interrupts the sleep.

Mode 1 — Dance

Zowi picks a random dance (IDs 5–20) and performs it 1–5 times. For dances 15–18 (bend, shakeLeg), it does 1 repetition at a slower tempo (1600 ms). For all others, it does 3–5 repetitions at 1000 ms.

Mode 2 — Obstacle Detection

Zowi walks forward until it detects an obstacle within 15 cm. When an obstacle is found, it shows surprise, jumps, looks confused, steps back, and checks again. If clear, it smiles and turns left. If still blocked, it repeats the reaction.

Mode 3 — Noise Detection

Zowi listens for noise (analog reading ≥ 650). When detected, it shows surprise, plays a sound, performs a random dance, and returns to home.

Mode 4 — Teleoperation

Activated automatically when serial data is received. Zowi disables button interrupts and processes serial commands via ZowiSerialCommand.

Key Functions

obstacleDetector()

Reads the ultrasonic sensor. If distance < 15 cm, sets obstacleDetected = true; otherwise false.

move(moveId)

Executes a movement by ID (0–20). Sends sendFinalAck() after each movement except for manual servo mode (ID ≥ 30).

ZowiLowBatteryAlarm()

If battery < 45%, flashes a thunder mouth and plays an alternating alarm tone (880 Hz ↔ 2000 Hz) until a button is pressed.

ZowiSleeping_withInterrupts()

Zowi leans forward (servos to 100, 80, 60, 120) and plays a 4-frame dream mouth animation with soft rising/falling tones. After 4 cycles, shows lineMouth and plays S_cuddly. Fully interruptible by button press.
